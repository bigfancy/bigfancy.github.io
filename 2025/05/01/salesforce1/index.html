gitea-sf運用

  • 開発
  1. 開発者がDEV Sandboxで作業
  2. DEV Sandbox → ローカルへ pull
  3. Giteaへ commit git checkout dev git add . git commit -m "initial commit" git push origin dev

  • テスト git checkout dev sfdx force:source:deploy -u test

  • 本番リリース git checkout main git merge dev git push origin main

merge request

sfdx force:source:deploy -u Production

git tag release-2025-01 git push origin --tags

main：直接push禁止（管理者のみ） dev：開発者push可

  • DEV Sandboxを複数人で安全に回す運用 git checkout dev git pull origin dev git checkout -b feature/userA-api-fix

sfdx force:source:pull -u DevSandbox git status git add . git commit -m "外部API疎通対応" git push origin feature/userA-api-fix

develop にマージ git checkout dev git merge feature/userA-api-fix git push origin dev

DEV Sandboxを「正しい状態」に戻す sfdx force:source:deploy -u DevSandbox

DEV Sandbox = dev

衝突（コンフリクト）が起きたらどうする？ 結論：Gitで解決する。Sandboxで直さない。
